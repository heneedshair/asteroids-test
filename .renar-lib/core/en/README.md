# RENAR Core — Conceptual Overview

## Where to Start

If you are encountering RENAR for the first time and want to quickly grasp what the standard is about — **start here**. Core is a short overview document for a human reader: what RENAR is, why it exists, how it works at a high level, and who will find it useful.

Core contains **no** technical details: artifact frontmatter, lifecycle transitions, normative rules, validation schemas. Those details live in the full [RENAR Standard](../../standard/en/README.md) (15 chapters).

| Document | Time | Contents |
|---|---|---|
| [renar-core.md](renar-core.md) | ≤ 10 min | Conceptual overview: what / why / how / for whom / where to read next |

After Core, depending on your role:

- **If you are a human** — proceed to [guide/00 Quickstart](../../guide/en/00-quickstart.md) for practical onboarding, or follow the role-based routes from [renar-core.md](renar-core.md#routes-by-role).
- **If you are an AI agent** — go directly to the [Standard](../../standard/en/README.md). Core is intended for human reading and is redundant for an AI.

---

## When Core Is Enough

Core is an overview, not a conformance mode. RENAR conformance is assessed **only** against the full Standard (manifest §13.4, mandatory clauses §13.3). Core does not introduce an alternative normative model.
